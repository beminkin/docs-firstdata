First Data Tile

 

Background and purpose.

First Data Tile provides benefits for developers using Pivotal Cloud Foundry environment to make it easier to utilize Payeezy API services (https://developer.payeezy.com). Payeezy is important API platform which First Data provides for developers and merchants to process payments.

Pivotal Cloud Foundry is a Platform as a Service (PaaS) offering by Pivotal which allows seamless deployment of the applications within public or private clouds. The strength of Pivotal Cloud Foundry is in ability to make deployment very simple (just pushing application binaries to the environment). It also enables fast and agile development of applications on the cloud.

Cloud Foundry is available as public service (PWS – https://run.pivotal.io) as well as private on-site installation (it installs on AWS and VMWare environments).

 

Benefits for Developers

Cloud Foundry provides ability to expose various things (such as databases and messaging systems) as services on the cloud. Applications using these services can automatically obtain service credentials and start making necessary API calls. This is exactly the benefit that we’re trying to utilize with the First Data Tile. It gives users capability to automatically wire Payeezy credentials into applications running on Cloud Foundry. Developers can utilize 3 different types of plans using First Data Tile:

-          Anonymous plan

o   Provides default (anonymous) set of credentials

-          Existing plan

o   Provides developers ability to enter set of credentials before requesting the tile

-          New Plan

o   Provides developers ability to request new set of credentials using their name and email

 

Installation of the Tile

Installation of the tile will be performed by Administrator of the particular Cloud Foundry environment. Tile is provided in .pivotal format which is acceptable by Cloud Foundry Ops Manager.

Currently, First Data Tile has been only installed in First Data internal test Cloud Foundry environment.

By default, Tile connects to Payeezy CERT environment, but it’s possible to specify other environments.

Security of the Tile.

First Data Tile relies on Pivotal Cloud Foundry security features. Credentials that are obtained by tile instances bound to the applications running on Cloud Foundry are stored securely and in encrypted form in Cloud Foundry internal database.

First Data Tile also uses master credentials to make calls to Payeezy for “New Plan” – to request new set of developer credentials. These credentials are currently hard coded in obfuscated format within the Tile code.

Calls made to request new credentials are made through Apigee and Drupal APIs provided by First Data Payeezy team. Standard Payeezy security is used (API key, Secret and HMAC calculation).

 

Usage outside of Cloud Foundry.

First Data Tile currently can run only on Pivotal Cloud Foundry.
